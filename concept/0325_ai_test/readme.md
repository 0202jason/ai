
# 📑 [심화 학습 노드] AI & 머신러닝 기초 이론 (1강)

## 1. AI, ML, DL의 정의 및 관계
강의에서는 세 개념을 포함 관계로 정의하며, 각 단계의 핵심 차이점을 강조합니다.

* **인공지능 (AI):** 가장 넓은 범주로, 환경이나 데이터를 인지·학습하여 목표 달성을 하도록 예측, 행동 선택, 플래닝(Planning)을 수행하는 시스템입니다.
* **머신러닝 (ML):** AI의 부분집합으로, 사람이 규칙을 직접 코딩(Hard Coding)하는 대신 **데이터로부터 규칙을 학습**하여 목적을 달성하는 접근 방식입니다.
* **딥러닝 (DL):** 머신러닝의 하위 분야로, **인공신경망(Neural Network)** 이라는 특정 함수 구조를 활용하여 학습을 수행하는 방법론입니다.
* **교수님의 Point:** 현대의 AI는 대부분 데이터를 기반으로 하는 머신러닝을 의미하며, 하드코딩된 규칙 기반 시스템(Rule-based)은 이번 강의의 범위를 넘어선 고전적 AI 방식입니다.

---

## 2. 데이터 구성 요소 (Data Components)
데이터는 모델이 학습하는 근거이자 정답의 쌍으로 이루어집니다.

* **피처 (Feature, $x$):** 모델 예측의 **판단 근거**이자 단서가 되는 입력 정보입니다.
    * *예시:* 이미지 분류 시 '이미지 자체', 이메일 스팸 분류 시 '메일 제목, 발신 주소, 단어 빈도' 등.
* **라벨 (Label/Target, $y$):** 모델이 예측하고자 하는 **정답 값**이며 학습의 목표값입니다.
* **데이터 분할 (Data Splitting):**
    * **훈련 데이터 (Train):** 모델을 적합(Fitting)시키고 규칙을 배우는 데 사용됩니다.
    * **검증 데이터 (Validation):** 학습 중 모델 성능을 가늠하고 최적의 설정을 찾는 용도입니다.
    * **테스트 데이터 (Test):** 학습에 전혀 사용되지 않은 **새로운 데이터**로 최종 일반화 성능을 확인합니다.

## 3. 회귀 (Regression) 및 결정계수 ($R^2$)
**회귀**: 입력(feature)으로부터 숫자(output)를 얼마나 정확히 예측하는가? 
**회귀**는 예측하려는 라벨이 **연속적인 수치(숫자)** 일 때 사용하는 지도 학습 기법입니다.

* **회귀의 핵심 원리:** 피처$(X)$와 라벨$(Y)$ 사이의 평균적인 관계를 나타내는 함수 $f$를 찾는 과정입니다.
* **평균제곱오차 ($MSE$)**: 실제 정답과 모델 예측값의 차이(오차)를 제곱하여 평균을 낸 값입니다.
  * **수식:**
  $$\text{MSE} = \frac{1}{n} \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$  
  * $y_i$: 실제 정답 (Label)  $\hat{y}_i$: 모델의 예측값  $n$: 데이터의 총 개수
* **결정계수 ($R^2$, R-squared):** 모델이 데이터의 변동성을 얼마나 설명하는지 나타내는 지표입니다.
  * **수식:** 
    $$R^2=1-\frac{\text{MSE(모델 오차)}}{\text{Variance(데이터 자체의 변동)}}$$  
    $$R^2 = 1 - \frac{\sum_{i=1}^{n} (y_i - \hat{y}_i)^2}{\sum_{i=1}^{n} (y_i - \bar{y})^2}$$
    $y_i$: 실제 정답 (Actual Value)  $\hat{y}_i$: 모델의 예측값 (Predicted Value)  $\bar{y}$: 실제 정답들의 평균값 (Mean Value)
  * **해석:** 1에 가까울수록 설명력이 높고, 0이면 평균으로만 예측하는 수준입니다. 음수가 나오면 평균보다 못한 성능을 뜻합니다.
* **비교:** MSE(평균 제곱 오차)는 **작을수록** 좋고, $R^2$는 **높을수록** 좋은 모델입니다.

| 항목 | **MSE** | **$R^2$** |
| :--- | :--- | :--- |
| **정의** | 오차의 제곱 평균 | 모델의 데이터 설명력 비율 |
| **값 범위** | $0 \sim \infty$ | $0 \sim 1$ |
| **단위** | 실제 데이터 단위의 제곱 | 단위 없음 (비율) |
| **해석** | 절댓값 (구체적 오차 크기) | 상대적 비율 (성능 정도) |
| **좋은 값** | **0**에 가까울수록 좋음 | **1**에 가까울수록 좋음 |
| **장점** | 구체적 오차 크기 알 수 있음 | 직관적 해석, 모델 간 비교 쉬움 |
| **단점** | 직관적 해석 어려움 | 구체적 오차 크기 모름 |

## 4. 혼동 행렬 (Confusion Matrix)
**분류** 문제에서 모델의 예측 결과를 평가하기 위해 작성하는 매트릭스입니다.

| 실제 \ 예측 | 예측 Positive | 예측 Negative |
| :--- | :---: | :---: |
| **실제 Positive** | **TP** (True Positive) | **FN** (False Negative) |
| **실제 Negative** | **FP** (False Positive) | **TN** (True Negative) |

* **불균형 데이터의 문제:** 실제 환자가 1%인 데이터에서 무조건 '정상'이라고만 예측해도 정확도는 99%가 나오기 때문에, 정확도 외의 지표가 반드시 필요합니다.

---

## 5. 분류 평가 지표 (Classification Metrics)
혼동 행렬을 기반으로 계산되는 핵심 지표들입니다.

1.  **정확도 (Accuracy):** 전체 중 맞춘 것의 비율. 
     $$\frac{TP + TN}{TP + TN + FP + FN}$$
2.  **정밀도 (Precision):** 모델이 Positive라고 예측한 것 중 진짜 Positive인 비율. 
   $$\text{Precision} = \frac{TP}{TP + FP}$$
   * 정밀도가 높으면 FP가 작음(Positive로 예측한 것 중 틀린게 적음. 소심하게 예측)
3.  **재현율 (Recall):** 실제 Positive인 것들 중 모델이 놓치지 않고 찾아낸 비율. 
   $$\text{Recall} = \frac{TP}{TP + FN}$$
   * 재현율이 높으면 FN이 작다(실제 Positive를 놓치지 않고 예측함)
4.  **F1-Score:** 정밀도와 재현율이 모두 높아야 높은 수치가 나오는 **조화 평균** 값입니다. 
   $$F1 = 2 \times \frac{\text{Precision} \times \text{Recall}}{\text{Precision} + \text{Recall}}$$

# 📑 [2강 심화 학습 노드] 신경망과 선형/로지스틱 회귀 기초

## 1. 잔차제곱합 RSS(Residual Sum of Squares)의 뜻과 정의
회귀 분석에서 모델이 얼마나 데이터를 잘 반영하지 못하는지(오차)를 측정하는 가장 기본적인 지표입니다.

* **정의:** 실제값($y_i$)과 모델이 예측한 값($\hat{y}_i$)의 차이인 **잔차(Residual)** 를 제곱하여 모두 더한 값입니다.
* **수식:** 
  $$RSS = \sum_{i=1}^{n} (y_i - \hat{y}_i)^2$$
* **의미:** RSS 값이 작을수록 모델이 학습 데이터를 잘 설명하고 있다는 뜻입니다. 머신러닝의 학습 목적은 이 RSS를 최소화하는 파라미터(가중치)를 찾는 것입니다.

---

## 2. 신경망 (Neural Network)
인간의 뇌 구조를 모방한 수학적 모델로, 딥러닝의 근간이 됩니다.

* **구성 요소:**
    * **입력층 (Input Layer):** 데이터(Feature)가 들어오는 곳.
    * **은닉층 (Hidden Layer):** 데이터의 복잡한 특징을 추출하는 곳. 층이 깊어질수록 'Deep' Learning이 됩니다.
    * **출력층 (Output Layer):** 최종 예측값(라벨)을 내보내는 곳.
* **작동 원리:** 각 연결선에는 **가중치(Weight)** 가 있으며, 입력값에 가중치를 곱하고 **편향(Bias)** 을 더한 뒤 **활성화 함수**를 거쳐 다음 층으로 전달됩니다.

---

## 3. 손실 함수 (Loss Function)
모델의 예측이 정답과 얼마나 틀렸는지 '벌점'을 주는 함수입니다.

* **정의:** 모델의 성능을 수치화한 함수로, 학습의 지표가 됩니다.
* **주요 종류:**
    * **MSE (Mean Squared Error):** 주로 회귀에서 사용. 오차의 제곱 평균.
    * **크로스 엔트로피 (Cross-Entropy):** 주로 분류(로지스틱 회귀 등)에서 사용. 확률 분포의 차이를 계산.
* **역할:** 모델은 학습 과정에서 이 손실 함수의 값을 최소화하는 방향으로 가중치를 업데이트합니다.

---

## 4. 경사하강법 (Gradient Descent)
손실 함수의 값을 최소화하기 위해 가중치를 점진적으로 업데이트하는 최적화(Optimization) 알고리즘입니다.

* **원리:** 현재 위치에서 손실 함수의 **기울기(Gradient)**를 구하고, 기울기가 낮아지는 방향으로 가중치를 조금씩 이동시킵니다.
* **수식:** $$w_{new} = w_{old} - \alpha \times \frac{\partial Loss}{\partial w}$$
    * $\alpha$ (Alpha): **학습률(Learning Rate)**. 너무 크면 건너뛰고, 너무 작으면 학습이 느려집니다.
* **비유:** 안개가 낀 산에서 가장 낮은 골짜기를 찾기 위해 발밑의 경사가 낮은 쪽으로 한 걸음씩 내려가는 것과 같습니다.

---

## 5. 로지스틱 회귀 (Logistic Regression)
이름은 회귀이지만, 실제로는 **분류(Classification)**를 수행하는 알고리즘입니다.

* **정의:** 선형 회귀의 출력값을 0과 1 사이의 확률값으로 변환하여, 데이터가 특정 클래스에 속할 확률을 예측합니다.
* **시그모이드 함수 (Sigmoid Function):** 어떤 숫자든 0~1 사이로 압축해주는 함수입니다. 로지스틱 회귀의 핵심입니다.
    * **수식:** $$\sigma(z) = \frac{1}{1 + e^{-z}}$$
* **결정 경계:** 확률이 0.5 이상이면 1(Positive), 0.5 미만이면 0(Negative)으로 분류합니다.

# 📑 [3강 심화 학습 노드] 자연어 처리(NLP)의 진화: RNN에서 BERT까지

## 1. RNN (Recurrent Neural Network) 원리 이해
RNN은 데이터의 **순서(Sequence)**가 중요한 텍스트, 음성, 영상 데이터를 처리하기 위해 고안된 모델입니다.

* **핵심 원리:** 이전 시점($t-1$)의 정보를 현재 시점($t$)의 학습에 반영하는 **재귀적 구조**를 가집니다.
* **은닉 상태 (Hidden State, $h_t$):** 과거의 정보를 압축해서 들고 있는 '메모리' 역할을 합니다.
* **수식:** $$h_t = \sigma(W_{xh}x_t + W_{hh}h_{t-1} + b)$$
* **파라미터 공유:** $W_{xh}, W_{hh}$는 모든 시점에서 **동일한 값**을 사용합니다.
* **한계:** 문장이 길어질수록 앞부분의 정보가 뒤로 전달되지 않는 **장기 의존성(Long-term Dependency)** 문제와 **기울기 소실(Vanishing Gradient)** 문제가 발생합니다.

---

## 2. LSTM (Long Short-Term Memory) 원리 이해
RNN의 치명적인 단점인 '장기 기억력 부족'을 해결하기 위해 등장한 모델입니다.

* **핵심 원리:** '셀 상태(Cell State)'라는 별도의 통로를 두고, **게이트(Gate)**를 통해 정보를 제어합니다.
* **3가지 주요 게이트:**
    1.  **망각 게이트 (Forget Gate):** 과거 정보 중 불필요한 것을 삭제.
    2.  **입력 게이트 (Input Gate):** 현재 정보 중 저장할 가치가 있는 것을 선택.
    3.  **출력 게이트 (Output Gate):** 다음 단계로 내보낼 값을 결정.
* **의의:** 중요하지 않은 정보는 거르고, 중요한 정보는 멀리까지 유지하여 긴 문장 처리 능력을 획기적으로 높였습니다.

---

## 3. Transformer 원리 이해
RNN처럼 순차적으로 읽지 않고, 문장 전체를 한꺼번에 처리하는 혁신적인 아키텍처입니다.

* **핵심 원리: 셀프 어텐션 (Self-Attention):** 문장 내 단어들끼리 서로의 관계를 계산하여, 어떤 단어가 문맥 파악에 중요한지 스스로 가중치를 둡니다.
* **장점:** 순차적 계산이 필요 없어 **병렬 연산**이 가능하며, 학습 속도와 성능이 압도적입니다.
* **구조:** 정보를 이해하는 **인코더(Encoder)**와 정보를 생성하는 **디코더(Decoder)**로 구성됩니다.
* **인코더 (Encoder):** 입력 문장을 읽고 그 맥락을 수치화된 벡터로 변환합니다. 여러 개의 동일한 층(Layer)이 쌓여 있으며, 각 층은 '셀프 어텐션'과 '피드 포워드 신경망'으로 구성됩니다.
* **디코더 (Decoder):** 인코더가 만든 정보와 이전에 생성한 단어들을 참조하여 다음 단어를 하나씩 예측합니다. 인코더와 달리 '인코더-디코더 어텐션' 층이 추가되어 인코더의 정보를 직접적으로 활용합니다.

### 셀프 어텐션 (Self-Attention)의 상세 계산 방식
셀프 어텐션은 문장 내의 각 단어가 서로 어떤 관련이 있는지 계산하는 과정입니다. 이를 위해 각 단어 벡터는 **Query(질문), Key(키), Value(값)**라는 세 가지 벡터로 변환됩니다.

#### ① Q, K, V 벡터 생성
입력된 각 단어 임베딩 벡터($x$)에 학습 가능한 가중치 행렬($W^Q, W^K, W^V$)을 곱하여 생성합니다.
* **Query ($Q$):** 현재 정보를 찾고자 하는 주체 (질문)
* **Key ($K$):** Query와 비교 대상이 되는 정보 (인덱스)
* **Value ($V$):** 실제 단어가 가진 의미 정보 (내용물)

#### ② 어텐션 스코어 (Attention Score) 계산
Query와 모든 Key 사이의 유사도를 내적(Dot-product)으로 구합니다.
$$\text{Score} = Q \cdot K^T$$

#### ③ 스케일링 (Scaling) 및 소프트맥스 (Softmax)
수치가 너무 커지는 것을 방지하기 위해 Key 벡터 차원($d_k$)의 제곱근으로 나누고, 합이 1이 되도록 소프트맥스를 취합니다. 이것이 각 단어에 대한 **'가중치(어텐션 가중치)'**가 됩니다.
$$\text{Attention Weight} = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)$$

#### ④ 최종 결과 추출
구해진 가중치를 Value($V$) 벡터에 곱하고 모두 더하여 최종 벡터를 산출합니다.
$$\text{Attention}(Q, K, V) = \text{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

---

## 4. BERT (Bidirectional Encoder Representations from Transformers) 원리 이해
트랜스포머의 인코더를 활용하여 문맥을 양방향으로 깊게 이해하도록 설계된 모델입니다.

* **핵심 원리:** 문맥을 왼쪽에서 오른쪽, 오른쪽에서 왼쪽 **양방향(Bidirectional)**으로 파악합니다.
* **주요 사전 학습(Pre-training) 전략:**
    1.  **MLM (Masked Language Model):** 문장의 일부 단어를 숨기고(Mask), 주변 문맥을 통해 그 단어를 추론합니다.
    2.  **NSP (Next Sentence Prediction):** 두 문장이 논리적으로 이어지는지 맞히는 학습입니다.
* **특수 토큰:** 문장 전체의 의미를 대표하는 `[CLS]` 토큰과 문장을 구분하는 `[SEP]` 토큰을 활용합니다.

---

## 5. NLP 성능평가 (Evaluation Metrics)
모델의 언어 처리 능력을 측정하는 대표적인 방법들입니다.

* **BLEU (Bilingual Evaluation Understudy):** 주로 기계 번역에서 사용하며, 정답 문장과 모델 생성 문장의 단어(n-gram)가 얼마나 일치하는지 측정합니다.
* **GLUE (General Language Understanding Evaluation):** 감성 분석, 문장 유사도 등 여러 태스크를 통해 모델의 범용적인 언어 이해 능력을 평가하는 벤치마크입니다.
* **정확도(Accuracy) 및 F1-Score:** 분류 문제에서 예측 성능을 측정할 때 주로 사용됩니다.

# 📑 [4강 심화 학습 노드] LLM의 추론과 평가, 그리고 인간 피드백 학습

## 1. 디코딩 알고리즘 (Decoding Algorithms)
언어 모델이 학습을 마친 후, 실제로 다음 단어를 어떻게 선택하여 문장을 완성할지 결정하는 전략입니다.

* **그리디 서치 (Greedy Search):** 각 시점에서 가장 확률이 높은 단어 하나만 선택합니다. 계산은 빠르지만, 전체 문장의 맥락에서 최선의 결과가 아닐 수 있으며 반복적인 문구가 생성될 위험이 있습니다.
* **빔 서치 (Beam Search):** 매 순간 상위 $k$개의 유망한 단어 조합(빔)을 유지하며 탐색합니다. 그리디 서치보다 더 높은 품질의 문장을 만들 수 있어 기계 번역 등에 자주 쓰입니다.
* **샘플링 (Sampling) 기법:**
    * **Temperature Scaling:** 확률 분포를 조절합니다. 온도가 높으면 낮은 확률의 단어도 선택될 기회를 주어 '창의적'인 답변을 내놓고, 낮으면 '보수적'인 답변을 내놓습니다.
    * **Top-K / Top-p (Nucleus) Sampling:** 상위 $K$개 혹은 누적 확률이 $p$가 되는 지점까지만 단어 후보를 제한하여 엉뚱한 단어가 선택되는 것을 방지합니다.

---

## 2. 거대 언어 모델(LLM)의 평가 (Evaluation)
LLM은 정답이 딱 정해져 있지 않은 경우가 많아 평가가 매우 까다롭습니다.

* **벤치마크 데이터셋:**
    * **MMLU:** 수학, 역사, 법률 등 다양한 전문 지식을 측정합니다.
    * **GSM8K:** 초등 산수 수준의 문장제 문제를 해결하는 논리력을 측정합니다.
    * **HumanEval:** 파이썬 코드 작성 능력을 평가합니다.
* **벤치마크의 한계 (Data Contamination):** 인터넷 데이터를 학습하는 LLM 특성상, 평가 문제 자체가 학습 데이터에 포함되어 '암기'로 문제를 풀 수도 있다는 점이 큰 이슈입니다.
* **LLM-as-a-Judge:** 최근에는 GPT-4 같은 더 강력한 모델이 다른 모델의 답변을 평가하게 하는 방식을 많이 사용합니다.

---

## 3. 선호 학습 (Preference Learning / RLHF)
모델이 단순히 다음 단어를 맞히는 것을 넘어, **인간이 좋아하는 방식**으로 답변하도록 길들이는 과정입니다.

* **SFT (Supervised Fine-Tuning):** 사람이 직접 쓴 고품질의 답변 데이터를 모델에게 추가 학습시킵니다.
* **보상 모델 (Reward Model) 학습:** 동일한 질문에 대해 모델이 내놓은 여러 답변 중, 사람이 어떤 것을 더 선호하는지 순위를 매긴 데이터를 통해 '인간의 선호도'를 판별하는 모델을 따로 만듭니다.
* **PPO (Proximal Policy Optimization):** 보상 모델로부터 점수를 높게 받도록 모델을 강화학습(Reinforcement Learning) 시킵니다.
* **DPO (Direct Preference Optimization):** 복잡한 강화학습 과정 없이, 직접 선호 데이터를 이용해 모델을 최적화하는 최신 기법입니다. 강의에서는 최근 PPO보다 효율적인 DPO가 많이 쓰인다고 언급됩니다.

---

## 4. 할루시네이션 (Hallucination) 및 정렬 (Alignment)
* **할루시네이션:** 모델이 사실이 아닌 내용을 그럴듯하게 지어내는 현상입니다. 이를 줄이기 위해 외부 지식을 검색해 답변에 활용하는 **RAG(Retrieval-Augmented Generation)** 기술이 중요하게 다뤄집니다.
* **정렬 (Alignment):** 모델의 목표를 인간의 가치관, 윤리, 의도와 일치시키는 과정입니다. 선호 학습이 이 정렬의 핵심 도구입니다.

# 📑 [5강 심화 학습 노드] CNN과 영상 모델의 이해

## 1. CNN (Convolutional Neural Network) 원리 이해
CNN은 이미지의 공간적 정보(위치, 형태)를 유지하며 특징을 추출하기 위해 고안된 신경망입니다.

* **핵심 동기:** 일반적인 완전 연결 계층(FC Layer)에 이미지를 넣으면 2차원 데이터를 1차원으로 펼치는 과정에서 **공간적 구조 정보가 손실**됩니다. CNN은 이를 보완하기 위해 '필터'를 사용하여 국소적인 특징을 찾아냅니다.
* **학습 대상:** CNN에서 학습되는 파라미터는 바로 **필터(커널)의 값**들입니다. 모델은 학습을 통해 어떤 필터를 써야 이미지의 특징을 가장 잘 뽑아낼 수 있는지 스스로 배웁니다.

---

## 2. 합성곱 연산 (Convolution Operation)
이미지 위를 필터(Filter)가 슬라이딩하며 대응하는 원소끼리 곱하고 그 합을 구하는 연산입니다.

* **연산 과정:** 입력 데이터와 필터의 각 요소를 곱한 뒤 모두 더하여 하나의 값을 산출합니다.
* **피처 맵 (Feature Map):** 합성곱 연산을 통해 만들어진 결과물로, 입력 이미지에서 특정 특징(선, 면, 질감 등)이 강조된 지도입니다.
* **수식적 이해:** 입력 $X$와 필터 $W$가 있을 때, 출력 $Y$의 한 점은 $\sum (X_{ij} \times W_{ij})$로 계산됩니다.

---

## 3. 스트라이드 (Stride)와 패딩 (Padding)
합성곱 연산 시 출력의 크기와 세밀함을 조절하는 중요한 파라미터입니다.

* **스트라이드 (Stride):** 필터를 한 번에 얼마나 이동시킬지 결정하는 '보폭'입니다.
    * 스트라이드가 커지면 출력 피처 맵의 크기는 작아집니다.
* **패딩 (Padding):** 입력 데이터의 외곽에 특정 값(보통 0)을 채우는 것입니다.
    * **목적:** 연산을 거칠수록 출력 크기가 작아지는 것을 방지하고, 이미지의 가장자리 정보를 충분히 반영하기 위해 사용합니다.

---

## 4. 풀링 (Pooling)
특징은 유지하면서 데이터의 차원(크기)을 줄여 연산량을 감소시키는 과정입니다.

* **주요 종류:**
    1.  **최대 풀링 (Max Pooling):** 설정된 영역 내에서 가장 큰 값만 선택합니다. (강한 특징 강조, 가장 많이 사용됨)
    2.  **평균 풀링 (Average Pooling):** 영역 내 값들의 평균을 구합니다.
* **특징:** 학습해야 할 파라미터가 없으며, 이미지의 미세한 위치 변화에 모델이 강인(Robust)해지도록 돕습니다.

---

## 5. CNN 연산 후 출력 크기 계산법 (시험 단골)
입력 크기($H, W$), 필터 크기($F$), 패딩($P$), 스트라이드($S$)가 주어졌을 때 출력 크기($OH, OW$)를 구하는 공식입니다.

$$\text{Output Size} = \frac{H + 2P - F}{S} + 1$$
*(이 값은 반드시 정수로 떨어져야 하며, 계산 문제로 자주 출제됩니다.)*

# 📑 [6강 심화 학습 노드] 멀티모달 AI와 파운데이션 모델의 활용

## 1. AI 파운데이션 모델 (Foundation Models)
특정 작업에 국한되지 않고, 방대한 데이터를 바탕으로 학습되어 다양한 하위 작업(Downstream Tasks)에 범용적으로 적용 가능한 모델을 의미합니다.

* **특징:** 하나만 잘하는 모델이 아니라, 요약, 번역, 코딩, 이미지 이해 등 **멀티태스킹**이 가능합니다.
* **대표 예시:** GPT 시리즈(텍스트), Stable Diffusion(이미지), CLIP(멀티모달) 등이 있습니다.
* **교수님의 Point:** 이제는 모델을 처음부터 만드는 것보다, 잘 만들어진 파운데이션 모델을 가져와 내 목적에 맞게 '적응(Adaptation)'시키는 능력이 더 중요해졌습니다.

---

## 2. Vision-Language Models (VLM)
텍스트와 이미지라는 서로 다른 형태(Modality)의 데이터를 동시에 이해하고 처리하는 모델입니다.

* **작동 원리:** 이미지를 설명하는 텍스트와 실제 이미지를 같은 의미 공간(Embedding Space)에 매핑하여, "사과"라는 단어와 사과 사진이 서로 가깝게 위치하도록 학습합니다.
* **주요 활용:** 이미지 캡셔닝(설명 생성), 시각적 질의응답(VQA), 이미지 검색 등이 있습니다.

---

## 3. SigLIP (Sigmoid Language-Image Pre-training)
구글에서 제안한 최신 멀티모달 학습 방식으로, 기존 CLIP 모델의 성능과 효율성을 크게 개선한 모델입니다.

* **핵심 차이점:** 기존 CLIP은 소프트맥스(Softmax)를 사용해 대조 학습을 했으나, SigLIP은 **시그모이드(Sigmoid) 손실 함수**를 사용하여 더 효율적인 학습이 가능하게 했습니다.
* **장점:** 대규모 배치 사이즈에서도 안정적으로 학습되며, 이미지와 텍스트 간의 매칭 성능이 매우 뛰어납니다. 현재 많은 최신 VLM의 뼈대로 사용되고 있습니다.

---

## 4. 이미지 생성 모델 (Image Generation Models)
텍스트 설명을 입력하면 그에 맞는 이미지를 새로 만들어내는 모델입니다.

* **확산 모델 (Diffusion Models):** 데이터에 노이즈를 섞었다가 다시 제거하는 과정을 반복하며 이미지를 생성합니다. (예: Stable Diffusion, DALL-E 3)
* **Autoregressive Models:** 텍스트를 생성하듯 이미지 픽셀 혹은 토큰을 순차적으로 생성하는 방식입니다.
* **교수님의 Point:** 생성 모델은 단순히 그림을 그리는 것을 넘어, 영상 편집이나 디자인 시안 제작 등 산업 전반으로 확장되고 있습니다.

---

## 5. 허깅페이스 (Hugging Face)
전 세계 AI 개발자들이 모델, 데이터셋, 데모 등을 공유하는 가장 대표적인 오픈소스 플랫폼입니다.

* **Transformers 라이브러리:** BERT, GPT, ViT 등 수많은 최신 모델을 단 몇 줄의 코드로 불러와 사용할 수 있게 해줍니다.
* **Model Hub:** 수십만 개의 사전 학습된(Pre-trained) 모델이 공유되어 있어, 누구나 최신 AI 기술을 쉽게 접할 수 있습니다.
* **Space:** 모델을 웹상에서 바로 실행해 볼 수 있는 데모 페이지를 제공합니다.
